Patrones de Diseño - Proyecto

Haz clic en el nombre del patrón para ver su carpeta con ejemplos.

Factory

Define una interfaz para crear un objeto, pero permite a las subclases decidir qué clase instanciar.

Ver carpeta Factory

Abstract Factory

Proporciona una interfaz para crear familias de objetos relacionados sin especificar sus clases concretas.

Ver carpeta Abstract Factory

Builder

Separa la construcción de un objeto complejo de su representación, permitiendo construir diferentes representaciones.

Ver carpeta Builder

Singleton

Garantiza que una clase tenga una única instancia y proporciona un punto de acceso global a ella.

Ver carpeta Singleton

Prototype

Permite copiar objetos existentes sin depender de sus clases concretas.

Ver carpeta Prototype

Adapter

Permite que interfaces incompatibles trabajen juntas convirtiendo una interfaz en otra esperada por el cliente.

Ver carpeta Adapter

Bridge

Separa una abstracción de su implementación para que ambas puedan variar independientemente.

Ver carpeta Bridge

Composite

Compone objetos en estructuras de árbol para representar jerarquías parte-todo.

Ver carpeta Composite

Decorator

Agrega responsabilidades adicionales a un objeto de manera dinámica.

Ver carpeta Decorator

Facade

Proporciona una interfaz simplificada a un conjunto complejo de subsistemas.

Ver carpeta Facade

Flyweight

Usa el compartir para soportar grandes cantidades de objetos de forma eficiente.

Ver carpeta Flyweight

Proxy

Proporciona un sustituto o representante de otro objeto para controlar el acceso a él.

Ver carpeta Proxy

Chain of Responsibility

Permite pasar solicitudes a lo largo de una cadena de manejadores hasta que uno la procese.

Ver carpeta Chain of Responsibility

Command

Encapsula una solicitud como un objeto, lo que permite parametrizar clientes con diferentes solicitudes.

Ver carpeta Command
